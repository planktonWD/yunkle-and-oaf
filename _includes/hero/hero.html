Official Styleguide

Please install grunt, Jekyll and sass. Then grab yourself a copy of this styleguide, cd to the directory and type grunt serve [enter] - Boom! Sass and JS compiled and SVG sprite made on the fly, plus a little live reload if you have installed. Just go to http://127.0.0.1:4000/formTab-styleguide/ (check the casing of Formtab-Syleguide) when you have run the command

Skinning

If you look in the Gruntfile.js you will see that at the time of writing this SASS is compiling two .css files from two major .scss files these are formtab.scss and eforms.scss

If you look at the formtab.scss you will see it imports a settings and function file along with the style file.

So create a new skin there are a couple of considerations. The site actually uses the site name as the variable for grabbing the correct stylesheet so name your master .scss file the name of the website. Duplicate eforms.scss, and create a new skin in the settings folder (maybe duplicate _eforms-skin.scss and rename it to _acme-skin.scss and correctly replace it in acme.scss.

Then open up Gruntfile.js and go to line 100 or so, you will see that eforms.scss is compiling to eforms.css so you will need to add a new line for acme.scss to acme.css

With that done you will now get a complied acme.css file with the acme skin applid to it. Just note that you will need to open in the styleguide site /includes/styleguide-head.html and make sure it is pulling in the right .css file so you can start tweaking the colors and testing etc

At the end make sure you upload all .css files in the assets folder to be sure that any global css changes are being applied to all sites.
